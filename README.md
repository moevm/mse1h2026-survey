# mse1h2026-survey

Веб-приложение для создания, прохождения и администрирования опросов.

## Запуск

1. Склонировать репозиторий:

```bash
git clone https://github.com/moevm/mse1h2026-survey.git
```

2. Перейти в каталог проекта:

```bash
cd mse1h2026-survey
```

3. Запустить контейнеры:

```bash
docker compose up --build
```

Backend автоматически применяет миграции Alembic при старте приложения. Если нужно применить миграции вручную:

```bash
docker compose exec backend alembic upgrade head
```

## Прохождение опроса

Опрос открывается по ссылке с ID:

```text
http://localhost:8080/survey/{id}
```

Например:

```text
http://localhost:8080/survey/1
```

На первой странице нужно ввести номер группы, например `3341` или `1303`, затем нажать кнопку начала опроса.

В админке на карточке опроса есть кнопка копирования ссылки на опрос.

## Администрирование

После входа в админку можно:

- просматривать список опросов;
- создавать новый опрос;
- редактировать существующий опрос;
- включать и выключать активность опроса;
- удалять опросы;
- копировать ссылку на прохождение опроса.

## Шаблонные вопросы и расписание

Шаблонные вопросы поддерживают подстановки:

```text
{{teacher}}
{{group}}
{{subject}}
```

При прохождении опроса приложение берет номер группы из URL-параметра и запрашивает данные расписания через:

```text
GET /api/group_data/{group}
```

Данные расписания хранятся в таблицах базы данных:

- `groups`
- `teachers`
- `disciplines`
- `group_teacher_disciplines`

## Импорт расписания из Google Sheets

Расписание можно импортировать из Google Sheets. Ссылка сохраняется через:

```text
POST /api/set_google_sheets_link
```

Импорт выполняется через:

```text
POST /api/import_from_sheets
```

Парсер находится в:

```text
backend/utils/parser.py
```

Он скачивает Google Sheets как `.xlsx`, разбирает листы и заполняет таблицы расписания в базе данных.

## Проверка сохраненных ответов

Открыть Swagger:

```text
http://localhost:8080/api/docs
```

Авторизоваться как администратор и выполнить запрос:

```text
GET /survey/answers/{id}
```

где `id` — ID опроса.
