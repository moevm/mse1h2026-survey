# Документация по опросам

## Основные страницы

Административная часть доступна после авторизации:

```text
http://localhost:8080/login
http://localhost:8080/dashboard
```

Публичное прохождение опроса открывается по ссылке с идентификатором опроса:

```text
http://localhost:8080/survey/{id}
```

На стартовой странице участник вводит номер группы. После этого приложение открывает прохождение опроса и использует группу для подстановки данных в шаблонные вопросы.

## Создание и редактирование

В админ-панели можно создать новый опрос или открыть существующий в редакторе. Для опроса настраиваются название, описание, активность, время жизни, группы, вопросы и при необходимости ссылка на Google Sheets с расписанием.

Поддерживаются обычные вопросы и шаблонные вопросы. Обычный вопрос отображается один раз. Шаблонный вопрос разворачивается в несколько вопросов по данным группы, преподавателей и дисциплин.

Для каждого вопроса обязательность задается отдельно. Если вопрос необязательный и пользователь оставил его пустым, он не должен блокировать отправку опроса.

## Шаблонные теги

Шаблонные вопросы используют теги в фигурных скобках:

```text
{{group}}
{{teacher}}
{{subject}}
```

Теги формируются на основе данных таблицы расписания. Если в таблице есть повторяющиеся названия столбцов, frontend должен отображать их различимо, чтобы администратор понимал, какой столбец будет использован.

В одном вопросе используется только один шаблонный тег. Это упрощает разворачивание вопросов и статистику по ответам.

## Логика teacher/subject

Порядок шаблонных вопросов влияет на набор вариантов:

- если сначала идет вопрос с `{{teacher}}`, а затем вопрос с `{{subject}}`, для выбранного преподавателя показываются только его предметы;
- если сначала идет вопрос с `{{subject}}`, а затем вопрос с `{{teacher}}`, для выбранного предмета показываются только преподаватели, которые его ведут.

Такой сценарий нужен, чтобы участник не видел нерелевантные пары преподаватель-предмет.

## Google Sheets

Для импорта расписания администратор указывает ссылку на Google Sheets. Backend скачивает таблицу, разбирает ее и сохраняет данные в таблицы расписания.

Основные endpoint-ы:

```text
POST /api/set_google_sheets_link
POST /api/import_from_sheets
GET /api/group_data/{group}?survey_id={survey_id}
```

Параметр `survey_id` нужен, чтобы данные группы подтягивались именно для текущего опроса.

## Статистика

Статистика по опросу доступна администратору. В ней должны учитываться реальные ответы, количество участников и группировка шаблонных вопросов.

Если несколько развернутых шаблонных вопросов относятся к одной логической связке, например `subject -> teacher` или `teacher -> subject`, в статистике они отображаются как один сгруппированный блок.

## API-документация

Swagger доступен по адресу:

```text
http://localhost:8080/api/docs
```

Через Swagger можно проверить административные ручки, ответы по опросу и импорт расписания.
