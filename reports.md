# mse-template

## О проекте

### Постановка задачи

Система кафедральных опросов — это веб-приложение для проведения опросов, хранения результатов и анализа полученных данных.

Основные цели системы:

- проведение опросов через веб-интерфейс
- хранение результатов опросов в базе данных
- просмотр результатов опросов
- анализ данных
- визуализация статистики
- экспорт результатов

Система предоставляет:

- пользовательский интерфейс для прохождения опросов
- административную панель для управления опросами
- REST API для взаимодействия frontend и backend
- хранение данных в PostgreSQL

Проект является развитием существующей системы:

https://survey.moevm.info/

---

### Требования

Основные функциональные требования:

Система должна обеспечивать:

- создание опросов
- редактирование опросов
- удаление опросов
- прохождение опросов по уникальной ссылке
- хранение ответов
- просмотр статистики
- экспорт данных
- визуализацию данных

Типы вопросов:

- **free** — свободный ответ
- **checkbox** — множественный выбор
- **radiobutton** — один вариант ответа
- **scale** — оценка от 1..N

Технологии:

**Backend**

- FastAPI
- SQLAlchemy
- PostgreSQL
- Pydantic

**Frontend**

- React
- JavaScript

**Infrastructure**

- Docker
- nginx

Нефункциональные требования:

- поддержка не менее 100 одновременных пользователей
- время ответа API не более 5 секунд
- безопасная авторизация пользователей
- хранение паролей в зашифрованном виде
- модульная архитектура

---

### Сценарии использования

#### Прохождение опроса

1. Пользователь открывает ссылку вида:

`/survey/{survey_id}`

2. Система проверяет статус опроса.

3. Если опрос активен:

- отображается название
- отображается описание
- отображаются вопросы

4. Пользователь отвечает на вопросы.

5. Пользователь отправляет ответы.

6. Система сохраняет ответы в базе данных.

7. Отображается сообщение об успешной отправке.

---

#### Управление опросами (Администратор)

1. Администратор входит в систему.

2. Администратор открывает административную панель.

3. Администратор может:

- создать опрос
- редактировать опрос
- удалить опрос
- включить опрос
- отключить опрос

---

#### Просмотр статистики

1. Администратор открывает страницу статистики.

2. Система отображает:

- количество ответов
- список вопросов
- распределение ответов

3. Система строит графики.

---

#### Запуск

1. Склонировать репозиторий:

```bash
git clone https://github.com/moevm/mse1h2026-survey.git
```

2. Перейти в каталог проекта:

```bash
cd mse1h2026-survey
```

3. Запустить контейнеры:

```bash
docker compose up --build
```

Backend автоматически применяет миграции Alembic при старте приложения. Если нужно применить миграции вручную:

```bash
docker compose exec backend alembic upgrade head
```

По результатам четвертой итерации система поддерживает полный основной сценарий работы с кафедральными опросами:

1. Администратор авторизуется и создает опрос.
2. При необходимости подключает Google Таблицу с расписанием.
3. Система получает доступные группы и шаблонные поля таблицы.
4. Администратор добавляет обычные и шаблонные вопросы, задает обязательность и порядок вопросов.
5. Для опроса формируется публичная ссылка с UUID.
6. Студент открывает ссылку, вводит свою группу и проходит доступный ему опрос.
7. Шаблонные вопросы автоматически раскрываются с учетом преподавателей и дисциплин его группы.
8. Ответы сохраняются в базе данных.
9. Администратор просматривает статистику и выгружает результаты в PDF или Excel.

Дополнительно в приложении реализованы черновики, обработка недоступных опросов, контроль доступа по ролям и автоматические CI-проверки.

### Задачи на следующую итерацию (Итерация 5)

В дальнейшей работе могут быть реализованы следующие улучшения:

#### Аналитика и отчеты

- расширенная фильтрация статистики по группе, преподавателю, дисциплине и периоду;
- дополнительные графики и аналитические показатели;
- настройка состава данных, включаемых в экспорт.

#### Пользователи и доступы

- регистрация и управление учетными записями пользователей;
- интеграция с внешней системой авторизации или LDAP;
- расширение ролевой модели доступа.

#### Качество и сопровождение

- расширение покрытия API и frontend-сценариев автоматическими тестами;
- обработка ошибок при некорректной или недоступной Google Sheets-ссылке;
- мониторинг, централизованное логирование и аудит действий администратора;
- оптимизация запросов к базе данных и производительности приложения.
